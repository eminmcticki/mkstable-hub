# mkstable-hub

## 项目简介

mkstable-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。项目本身不对应任何特定域名或具体网站，仅作为静态 HTML 内容的集合与分发点。

本仓库主要面向需要长期保存、版本化管理的 HTML 页面，适合用作小型文档、工具页面、演示页面或轻量级前端的存档。

## 目录说明

```
mkstable-hub/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 公共资源文件（样式、脚本、图片等）
├── archive/        # 历史版本或已归档的页面
└── README.md       # 本文件
```

- `pages/`：主要目录，每个 HTML 页面为一个独立文件或子目录。
- `assets/`：存放页面间共享的资源文件，如 CSS、JS、图标等。
- `archive/`：用于存放不再更新但需保留的旧版本页面。

## 页面归档说明

- 所有页面均为纯前端静态 HTML，不依赖后端服务。
- 页面之间彼此独立，无功能耦合。
- 归档时请保持文件结构清晰，建议为每个页面创建单独文件夹（如 `pages/example-page/`），内含 `index.html` 及所需资源。
- 若页面引用公共资源，请确保路径正确，优先使用相对路径。

## 维护说明

- 欢迎提交 Pull Request 添加新的 HTML 页面或改进现有页面。
- 请确保新增页面不包含恶意代码、外部追踪脚本或侵犯第三方权益的内容。
- 建议为每个页面添加简要说明（可在页面头部注释或单独 `README.md` 中注明）。
- 历史归档文件请勿随意删除，如需废弃请移入 `archive/` 目录。
